# Dashboard UX Patterns — 금융 대시보드 UX 패턴

## 워크플로우

### 1. 정보 위계 4단계 설계
- L1 (영구 노출): 레짐 신호등 + 활성 critical 알림 배너
- L2 (스크롤 없이): 핵심 6~8개 지표 카드
- L3 (스크롤/탭): 보조 지표, 상세 차트
- L4 (드릴다운): 개별 지표 상세 페이지

### 2. 위젯 타입 선택 가이드

| 데이터 특성 | 권장 위젯 | 예시 지표 |
|-----------|---------|---------|
| 단일 수치 + 상태 | 신호등 카드 | VIX, CPI YoY |
| 시계열 추세 | 스파크라인 + 큰 숫자 | 10Y 금리 |
| 0 기준 양/음 | 스프레드 차트 | 10Y-2Y |
| 카테고리별 상태 | 신호등 히트맵 | 전체 카테고리 한눈에 |
| 시간순 이벤트 | 알림 피드 | 발생한 시그널 목록 |
| 분류 결과 | 레짐 배지 | 현재 거시 환경 |
| 다지표 비교 | 정규화 라인차트 | VIX vs HY 스프레드 |

### 3. 색상 코딩 표준 (색맹 고려)

| 상태 | 색상 (HEX) | Tailwind | 아이콘 |
|------|-----------|---------|-------|
| 정상 (normal) | #10b981 | green-500 | ● |
| 경계 (watch) | #f59e0b | amber-500 | ▲ |
| 위험 (alert) | #ef4444 | red-500 | ⚠ |
| 스테일 | #6b7280 | gray-500 | — |
| 매수 후보 | #3b82f6 | blue-500 | ★ |

색상만이 아닌 아이콘/패턴 병행 필수.

### 4. 반응형 브레이크포인트

```
Desktop  ≥1280px → 12 columns, cards 4×3
Tablet   768~1279 → 8 columns, cards 2×N
Mobile    <768px  → 1 column, cards stacked (신호등은 sticky top)
```

### 5. 알림 표시 패턴

- **인앱 배너**: critical 알림은 상단 sticky 배너 (사용자 dismiss 가능)
- **알림 피드**: 메인 화면 하단 또는 별도 패널, 시간 역순
- **카드 상태 변화**: 임계치 넘는 순간 카드 테두리/배경에 펄스 애니메이션
- **푸시 옵션**: 웹 푸시 (브라우저 권한 필요), 또는 Slack/이메일 외부 채널

### 6. 데이터 신선도(Freshness) 표시

- 마지막 갱신 시각 항상 표시
- 정해진 갱신 주기를 넘기면 "stale" 배지
- 데이터 없음(API 실패 등)은 "—" + 회색 처리
